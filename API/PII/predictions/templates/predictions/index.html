Récolte des données

Pour la récupération des données, nous avons travaillé sur plusieurs possibilités non-concluantes : Twitter (aucune information dans la majoritée des tweets), Bitcoin.fr (pas assez de contenu), stocktwits (API payante et énorme difficulté pour scrapper le site)… Finalement, nous avons choisi de collecter la donnée du site Reddit. Reddit avait l’avantage de proposer un contenu riche, qualitatif et une API gratuite et riche en possibilité. Toute la donnée récupérée a été stockée par date dans des csv.

Nettoyage des données

Pour le nettoyage des données, nous avons testé énormément de package pour nous permettre de réduire la taille des csv tout en supprimant la data inutile qui risquait de fausser les résultats. Les packages utilisés sont les suivants :
- Stemming et Lemmatize, pour regrouper les mots de la même famille
- des dictionnaires, pour supprimer les mots trop et peu présents car ils n’apportent aucunes informations pour différencier les jours
- Unigram Tagger : Pour garder seulement les types de mots que l’on veut et qui transmettent une émotion
- Manuel : Suppression des commentaires ayant des mauvaises appréciations,…

Traitement des données

Nous avons traité les données pour faire ressortir seulement les éléments importants de chaque post/journée :
- NLTK et Vader : afin de ne garder que les mots qui transmettent une émotion
- LDA dont le but est de faire ressortir les sujets principaux de chaque journée
- TF-IDF qui fait apparaitre les mots qui différencient un texte d’un autre

Prédiction

Pour la prédiction du modèle, nous avons confronté un algorithme de Deep Learning avec un algorithme de Machine Learning : Keras/Xgboost. Nous avons réalisé plusieurs modèles et des fonctions de paramétrage automatique pour obtenir les meilleurs résultats. En particulier, pour le modèle de Deep Learning pour la création des couches de neurones.

Résultats

Model Précision MSE
Texte 26% 2.69
Date 33% 1.54
Naïf (toujours 3) 25% 1.96
Malgré les tentatives d’optimisation au maximum de l’information contenue dans les messages récupérées depuis les réseaux sociaux, nous n’avons jamais réussi à obtenir des résultats probants et nous n’avons jamais trouvé une corrélation significative. De plus, nous avons réalisé des tests en prédisant le cours de la journée en fonction, uniquement, des cours de la veille, de l’avant-veille et la date du jour (voir ligne date) et nous obtenons des meilleurs résultats. De plus, nous pouvons voir que notre modèle obtient des résultats similaires au modèle naïf.
La conclusion de notre projet est qu'il n'y a pas de corrélation entre les messages sur les réseaux sociaux et le cours du Bitcoin

Site Web

Pour le site web, nous avons travaillé sur la réalisation d’une API Python avec le package Django pour pouvoir afficher nos résultats et nous avons avons réalisé un site web esthétique et simple pour comprendre les étapes de notre projet et mieux voir les résultats obtenus.
